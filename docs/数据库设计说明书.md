# 📘 基于泛雅平台的 AI 互动智课系统 - 数据库设计说明书

## 📖 文档概述
本数据库 Schema 专为满足 **“A12：基于泛雅平台的 AI 互动智课生成与实时问答系统”** 赛题设计。架构紧扣赛题执行手册中的 **“闭合回路”** 要求，分为三个核心数据域：

- **基础信息域（Base Information Domain）**：对接平台权限与用户体系，支撑课程列表与多租户隔离
- **智课生成域（Smart Generation Domain）**：支撑课件解析与结构化引擎（CIR/节点树），构建“系统大脑”
- **互动与续接域（Interaction & Continuation Domain）**：支撑智能问答、证据溯源、断点续学与节奏适配，形成学习闭环

---

## 一、宏观数据流向：从“静态文件”到“动态闭环”
整个系统的数据流向是一条清晰的单行道，最后在“学生端”形成一个完美的闭环：

`教师/系统（users）` ➡️ `创建分类与课程（course_categories, courses）` ➡️ `上传课件（lessons）` ➡️ `系统异步拆解结构树（cir_sections）` ➡️ `学生（users）在会话中提问并产生可溯源证据（qa_records）与断点位置（learning_progress）` ➡️ `系统引导学生回到结构树继续学习`

---

## 版本说明（版本 B：字段命名与后端实现对齐）
为实现“最小改动路径”，本版本在保持表名不变的前提下，统一将教学结构锚点从 `section_id` 命名升级为 `node_id`，并补齐当前后端在会话、证据与学习状态上已使用的字段结构：

- 统一命名：`section_id` → `node_id`，`cited_section_id` → `cited_node_id`，`current_section_id` → `current_node_id`。
- 证据问答：在保留“主引用节点”的同时，新增 `sources`（多来源证据数组）、`bbox/image_url/page_num` 等前端高亮所需信息。
- 会话状态：在进度表中新增 `session_id/current_path/current_topic/last_qa_query/confusion_count/mastery` 以承接后端现有状态机与掌握度逻辑。
- 用户画像：扩展 `users` 表以支撑 UserProfile，提升 AI 对齐理解程度与节奏适配效果。

## 二、微观表关系解析（分层说明）

### 1. 基础底座层：权限与容器的挂载关系

**users ↔ courses（一对多）**  
表 `courses` 中的 `teacher_id` 字段指向 `users` 表。这明确了“谁是这门课的主讲人”，确保教师端只能管理和编辑自己的课程数据。

**course_categories ↔ courses（一对多）**  
表 `courses` 中的 `category_id` 指向 `course_categories` 表。这支撑了前端视频中展示的“高等数学”、“大学物理”等多级折叠菜单的渲染逻辑。

### 2. 核心引擎层：化整为零的拆解关系

**courses ↔ lessons（一对多）**  
一门完整的课程通常包含多个章节的 PPT。表 `lessons` 通过 `course_id` 挂载到具体课程下，作为物理文件（PPT/PDF）的直接管理者。

**lessons ↔ cir_sections（一对多，🌟系统核心）**  
当一个 PPT 文件完成解析后，它在数据库中就不再是一个单一的文件了。表 `cir_sections` 会通过 `lesson_id` 挂载数十条记录（代表该 PPT 的每一页或每一个知识点）。这种关系将“庞大的静态文件”打碎成了“灵活的知识卡片”。

**cir_sections ↔ cir_sections（自关联 / 无限层级）**  
表 `cir_sections` 内部包含一个 `parent_id` 字段，指向它自己。这种“自己关联自己”的设计，完美实现了赛题要求的“第一章 -> 第一节 -> 知识点页”这种深层级的目录树结构。

### 3. 互动闭环层：锚定与溯源关系
这一层的设计直接决定了赛题中“多轮问答”与“智能续讲”的成败。

**qa_records ↔ cir_sections（多对一，🌟得分关键）**  
每次学生提问产生一条 `qa_records` 记录时，不仅要记录 `user_id` 和 `lesson_id`，更核心的是必须通过 `cited_node_id` 关联到 `cir_sections` 中的具体某一条记录（主引用节点）。同时建议存储 `sources` 作为“多来源证据数组”，兼容后端检索返回的多个证据片段。这样既保留“答案来自第 X 页”的评测要点，又能支撑前端对多个证据位置的高亮展示。

**qa_records ↔ qa_records（会话串联）**  
通过 `session_id` 字段，将同一个学生对同一个知识点的连续多次追问（多条独立的问答记录）串联成一个完整的“多轮对话历史”。

**learning_progress ↔ cir_sections（一对一 / 定位指针）**  
每当学生打断视频进行提问时，系统会将当前正在讲授的节点 ID 更新到 `learning_progress` 表的 `current_node_id` 字段中，并记录 `session_id`、`last_position_seconds` 等断点信息。问答结束后，前端只需查询这些字段，就能像书签一样精准回到原来讲到的位置继续讲；同时结合 `qa_records` 的理解度与 `learning_progress.mastery` 的掌握度数据触发补讲/加速等节奏策略。

## 模块一：基础信息域 (Base Information Domain)
本模块负责对接外部平台（如泛雅）的用户体系，并构建课程的多级树状分类模型，支撑前端的课程列表渲染。

### 1. users（用户信息表）
**作用**：存储系统所有用户（教师、学生）的账户信息，支持手机号验证与角色鉴权保护。并提供 UserProfile（画像）能力，让用户补充个人背景信息以便 AI 更准确对齐理解程度与解释风格。

| 字段名 | 数据类型 | 约束 | 业务说明 |
| :--- | :--- | :--- | :--- |
| user_id | VARCHAR(50) | PRIMARY KEY | 平台用户的唯一标识（如 `stu20001`）。 |
| school_id | VARCHAR(50) | NOT NULL | 学校标识，用于多租户与机构数据隔离（必填）。 |
| grade | VARCHAR(20) | NOT NULL | 年级（必填，如“大一/2024级”）。 |
| major | VARCHAR(100) | NOT NULL | 专业（必填）。 |
| phone_number | VARCHAR(20) | UNIQUE | 手机号，匹配前端演示中的登录/注册账号。 |
| user_name | VARCHAR(100) | NOT NULL | 用户的真实姓名或昵称。 |
| role | VARCHAR(20) | CHECK 枚举 | 角色身份，限定为 `teacher` 或 `student`。 |
| contact_info | JSONB | - | 扩展字段，用于加密存储邮箱、微信等敏感联系方式。 |
| profile_text | TEXT | - | 选填：用户对自己的补充说明（学习基础、偏好、目标等）。 |
| profile_ext | JSONB | - | 选填：结构化画像扩展（兴趣、强弱项、偏好交互模式等）。 |
| created_at | TIMESTAMP | DEFAULT NOW() | 账户注册/同步的创建时间。 |

### 2. course_categories（课程顶级分类表）
**作用**：实现前端视频中“高等数学”、“大学物理”等大类的折叠菜单。

| 字段名 | 数据类型 | 约束 | 业务说明 |
| :--- | :--- | :--- | :--- |
| category_id | SERIAL | PRIMARY KEY | 分类自增主键。 |
| school_id | VARCHAR(50) | NOT NULL | 学校标识，用于多租户隔离。 |
| category_name | VARCHAR(100) | NOT NULL | 大类名称（如“高等数学”）。 |
| sort_order | INT | DEFAULT 0 | 排序权重，控制分类在前端显示的上下顺序。 |

### 3. courses（课程体系表）
**作用**：记录具体的教学课程单元，关联所属大类与授课教师。

| 字段名 | 数据类型 | 约束 | 业务说明 |
| :--- | :--- | :--- | :--- |
| course_id | VARCHAR(50) | PRIMARY KEY | 课程的唯一业务编号。 |
| school_id | VARCHAR(50) | NOT NULL | 学校标识，用于多租户隔离。 |
| category_id | INT | FOREIGN KEY | 关联 `course_categories` 表，标明所属大类。 |
| course_name | VARCHAR(200) | NOT NULL | 课程名称（如“第一章：极限基础”）。 |
| teacher_id | VARCHAR(50) | FOREIGN KEY | 关联 `users` 表，标识该课程的创建人/授课教师。 |
| term | VARCHAR(20) | - | 学期标识（如 `2024-Fall`）。 |
| ext_info | JSONB | - | 课程扩展信息，可灵活存储学时、学分、课程简介等。 |

## 模块二：智课生成域 (Smart Generation Domain) 🌟核心
系统的“大脑”。负责将静态的 PPT/PDF 文件拆解为机器可读、前端可展示的 **课程结构（CIR）**。

### 4. lessons（课件与解析任务表）
**作用**：课件的物理载体池，同时负责监控“异步解析任务”的状态与耗时，直接产出赛题要求的“2分钟/份”性能指标证据。

| 字段名 | 数据类型 | 约束 | 业务说明 |
| :--- | :--- | :--- | :--- |
| lesson_id | VARCHAR(50) | PRIMARY KEY | 课件及解析任务的唯一ID（即 API 中的 `parseId`）。 |
| course_id | VARCHAR(50) | FOREIGN KEY | 关联 `courses` 表，标明课件归属哪门课程。 |
| school_id | VARCHAR(50) | NOT NULL | 学校标识，用于多租户隔离。 |
| title | VARCHAR(200) | NOT NULL | 前端课程卡片上展示的标题（如“数学基础课程”）。 |
| cover_image | VARCHAR(500) | - | 视频中显示的 PPT 封面缩略图 URL。 |
| file_type | VARCHAR(10) | - | 文件格式（如 `ppt`, `pdf`）。 |
| file_url | VARCHAR(500) | NOT NULL | 原始课件的公网存储地址，供解析引擎下载。 |
| category | VARCHAR(20) | - | 文/理/工分类，用于结赛统计“100份样本分布情况”。 |
| task_status | VARCHAR(20) | DEFAULT | 异步任务状态：`processing` / `completed` / `failed`。 |
| created_at | TIMESTAMP | DEFAULT NOW() | 任务提交时间。 |
| completed_at | TIMESTAMP | - | 解析完成时间（`completed_at - created_at` 即单份解析耗时）。 |
| file_info | JSONB | - | 存储文件大小、总页数等物理属性。 |

### 5. cir_sections（CIR 课程中间结构表）
**作用**：本系统最具价值的数据表。它通过自包含的递归关系将一份大课件拆解成“章->节->页”的目录树，并映射所有音视频与文本素材。

| 字段名 | 数据类型 | 约束 | 业务说明 |
| :--- | :--- | :--- | :--- |
| node_id | VARCHAR(50) | PRIMARY KEY | 教学节点唯一ID（统一命名为 node）。 |
| lesson_id | VARCHAR(50) | FOREIGN KEY | 关联 `lessons` 表，归属某个课件。 |
| school_id | VARCHAR(50) | NOT NULL | 学校标识，用于多租户隔离。 |
| node_name | VARCHAR(200) | NOT NULL | 节点标题（前端目录树展示）。 |
| parent_id | VARCHAR(50) | FOREIGN KEY | 指向本表父节点ID（同为 node_id）；为空表示根节点，实现无限层级。 |
| node_type | VARCHAR(20) | CHECK 枚举 | `chapter` / `subchapter` / `point`（知识点物理页）。 |
| sort_order | INT | DEFAULT 0 | 同级节点展示顺序，防止播放大纲错乱。 |
| path | TEXT | - | 节点路径（如 `/chapter1/section2/point3`），用于会话对齐与检索 rerank。 |
| page_num | INT | - | 对应原始 PPT 物理页码（用于“答案来自第 X 页”展示）。 |
| image_url | VARCHAR(500) | - | 右侧面板展示的该页 PPT 截图地址。 |
| bbox | JSONB | - | 选填：答案高亮定位框（`[x, y, w, h]`，归一化 0~1）。 |
| key_points | JSONB | - | 大模型提取的本页核心知识点（3-7个）。 |
| teaching_content | TEXT | - | 结构化教学内容（教学步骤、课程目标等）。 |
| script_content | TEXT | - | 喂给语音合成引擎的讲授文本/播音稿。 |
| duration_seconds | INT | - | 预计讲授音频时长（秒）。 |
| audio_url | VARCHAR(500) | - | 由 `script_content` 合成的语音文件地址。 |
| media_resources | JSONB | - | 从该页提取的图片、公式、图表等结构化数据。 |
| is_teacher_edited | BOOLEAN | DEFAULT FALSE | 标记讲稿是否被教师人工修正。 |

## 模块三：互动与续接域 (Interaction & Continuation Domain)
实现赛题“打断提问 -> 证据溯源 -> 自动续接”学习闭环的关键追踪模块。

### 6. qa_records（问答交互记录表）
**作用**：记录所有互动数据，支持多轮对话上下文管理，并产出赛题要求的“5秒响应指标”与“引用溯源”证据。

| 字段名 | 数据类型 | 约束 | 业务说明 |
| :--- | :--- | :--- | :--- |
| answer_id | SERIAL | PRIMARY KEY | 问答记录自增ID。 |
| session_id | VARCHAR(50) | NOT NULL | 会话ID；相同会话代表同一话题的多轮追问。 |
| user_id | VARCHAR(50) | FOREIGN KEY | 提问学生ID。 |
| school_id | VARCHAR(50) | NOT NULL | 学校标识，用于多租户隔离。 |
| lesson_id | VARCHAR(50) | FOREIGN KEY | 所属课件ID（可空：允许纯会话问题或尚未绑定课件的阶段）。 |
| question_type | VARCHAR(10) | - | `text` / `voice`。 |
| question_text | TEXT | NOT NULL | 学生问题文本（语音识别后也落这里）。 |
| answer_text | TEXT | NOT NULL | AI 结合课件上下文生成的回答。 |
| cited_node_id | VARCHAR(50) | FOREIGN KEY | 主引用节点：关联 `cir_sections.node_id`（可空：允许多来源证据但暂未选主引用）。 |
| source_page_num | INT | - | 主引用页码（可空）。 |
| sources | JSONB | - | 多来源证据数组（每项包含 `node_id/path/page_num/image_url/bbox/relevance_score/content` 等）。 |
| current_path | TEXT | - | 提问发生时的学习路径，用于对齐 `cir_sections.path`。 |
| video_timestamp | DOUBLE PRECISION | - | 提问发生时的播放器时间戳（秒），用于问答结束后自动续接。 |
| understanding_level | VARCHAR(20) | - | `none` / `partial` / `full`，用于触发节奏调整。 |
| response_ms | INT | - | 响应耗时（毫秒），用于统计“5秒响应”达标率。 |
| is_accurate | BOOLEAN | - | 线下评测字段：人工抽检该回答是否准确，用于统计准确率。 |
| reasoning_content | TEXT | - | 选填：reasoner 的思考过程（用于复盘/评测/调试）。 |
| tool_calls | JSONB | - | 选填：工具调用记录（如 web_search/math_solver），用于复盘。 |
| created_at | TIMESTAMP | DEFAULT NOW() | 提问时间戳。 |

### 7. learning_progress（学习进度与节奏表）
**作用**：记录学生实时学习坐标；发生提问打断后依赖该表回到原处继续讲授，并基于理解程度调整节奏。

| 字段名 | 数据类型 | 约束 | 业务说明 |
| :--- | :--- | :--- | :--- |
| track_id | VARCHAR(50) | PRIMARY KEY | 学习追踪记录唯一ID。 |
| user_id | VARCHAR(50) | FOREIGN KEY | 关联学生用户。 |
| session_id | VARCHAR(50) | NOT NULL | 会话ID（与后端状态、历史、掌握度对齐）。 |
| school_id | VARCHAR(50) | NOT NULL | 学校标识，用于多租户隔离。 |
| lesson_id | VARCHAR(50) | FOREIGN KEY | 关联正在学习的课件（可空：允许先聊天后绑定课件）。 |
| current_node_id | VARCHAR(50) | FOREIGN KEY | 断点续播核心：当前停留的 `cir_sections.node_id`。 |
| current_path | TEXT | - | 当前学习路径（对齐 `cir_sections.path`）。 |
| current_topic | TEXT | - | 当前主题（用于掌握度与策略决策）。 |
| last_qa_query | TEXT | - | 最近一次有效问答问题（用于短问题追问的上下文对齐）。 |
| confusion_count | INT | DEFAULT 0 | 当前主题的困惑计数（触发补讲/回退）。 |
| mastery | JSONB | DEFAULT '{}' | 主题掌握度映射（如 `{ "极限": 0.73 }`）。 |
| last_position_seconds | INT | DEFAULT 0 | 断点续播核心：音频播放到第几秒（如打断发生在第15秒则记录15）。 |
| progress_percent | FLOAT | - | 总体学习进度（0.0 ~ 100.0），用于前端进度条。 |
| adjust_type | VARCHAR(20) | DEFAULT 'normal' | 节奏策略：`normal` / `supplement` / `accelerate`。 |
| needs_supplement | BOOLEAN | DEFAULT FALSE | 标志位：追问多次仍不懂，触发补讲。 |
| last_operate_time | TIMESTAMP | DEFAULT NOW() | 最后一次学习/互动操作时间。 |
