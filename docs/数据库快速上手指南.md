# 数据库快速上手指南

## 1. 环境准备

### 1.1 安装依赖
```bash
conda run -n ai_ppt_web pip install psycopg2-binary sqlalchemy alembic
```

### 1.2 确认数据库配置
配置文件：`backend/app/core/config.py`

## 2. 数据库迁移（首次使用）

```bash
cd backend

# 执行迁移（创建所有表）
conda run -n ai_ppt_web alembic upgrade head

# 查看当前版本
conda run -n ai_ppt_web alembic current
```

## 3. 快速开始

### 3.1 在 Service 中使用数据库

```python
from sqlalchemy.orm import Session
from backend.app.core.database import get_db
from backend.app.repositories import QARecordRepository, LearningProgressRepository

class YourService:
    def __init__(self, db: Session = None):
        self.db = db
        self.qa_repo = QARecordRepository(db) if db else None
        self.progress_repo = LearningProgressRepository(db) if db else None
```

```python
# 在生成回答后调用
```

## 5. 降级策略（重要）

数据库操作必须**不影响主业务流程**：

```python
# 正确做法：try-except 包裹所有 DB 操作
if self.qa_repo:
    try:
        self.qa_repo.create_record(...)
    except Exception as e:
        logger.error(f"DB error: {e}")
        # 失败时继续执行，不中断主流程

# 错误做法：没有异常处理
self.qa_repo.create_record(...)  # 如果失败会导致整个请求失败
```

## 6. 常见操作

### 6.1 检查数据库连接

```python

if check_db_connection():
    print("数据库连接正常")
```

### 6.2 直接执行 SQL（不推荐，但可用）

```python
from backend.app.core.database import SessionLocal

db = SessionLocal()
try:
    result = db.execute(text("SELECT * FROM users LIMIT 5"))
    for row in result:
        print(row)
finally:
    db.close()
```

## 7. 表结构速查

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| users | 用户信息 | user_id, user_name, role, school_id |
| courses | 课程信息 | course_id, course_name, teacher_id |
| lessons | 课件信息 | lesson_id, title, task_status |
| cir_sections | CIR节点 | node_id, node_name, node_type, path |
| qa_records | 问答记录 | answer_id, session_id, question_text, answer_text, sources |
| learning_progress | 学习进度 | track_id, user_id, session_id, mastery, confusion_count |

## 8. 故障排查

### 8.1 连接失败
- 检查网络是否能访问 10.0.0.4:5432
- 确认 config.py 中的 DATABASE_URL 配置正确
- 检查 PostgreSQL 服务是否运行

### 8.2 外键约束错误
- 确保引用的 user_id、lesson_id、node_id 在对应表中存在
- 使用数据库中已存在的数据进行测试

### 8.3 表不存在
- 执行 `alembic upgrade head` 创建表
- 检查迁移脚本是否正确生成
