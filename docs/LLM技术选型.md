

此方案旨在为 **FWWB A12 智能交互课程系统** 提供科学的 LLM 选型依据。鉴于本项目涉及 **RAG 知识问答、教学脚本生成、多轮对话互动** 等核心场景，且面向中文教育领域，选型将重点考量 **中文理解能力、长文本处理（RAG）、推理能力（教学引导）及成本效益**。

## 2. 选型决策矩阵 (Decision Matrix)

基于项目需求（教育/RAG/Python栈），建立如下评分体系（总分 100）：

| 维度          | 权重  | 评分标准                                      | 获胜者 (预测)                   |
|:----------- |:--- |:----------------------------------------- |:-------------------------- |
| **中文教学能力**  | 25% | 能够生成通顺、符合教育心理学的中文解释，准确理解理科术语。             | DeepSeek V3 / Qwen2.5      |
| **RAG 适配性** | 20% | 上下文窗口 (>32k)，指令遵循（不产生幻觉，严格基于 Context 回答）。 | Claude 3.5 / DeepSeek      |
| **成本效益**    | 20% | 百万 Token 价格（API）或 硬件摊销成本（自建）。             | **DeepSeek V3** (绝对优势)     |
| **响应速度**    | 15% | TTFT (首字时间) < 1s，生成速度 > 50 tokens/s。      | GPT-4o / Qwen-Turbo        |
| **开发集成度**   | 10% | OpenAI SDK 兼容性，Function Calling 支持。       | GPT-4o / DeepSeek          |
| **合规与安全**   | 10% | 数据不离境，符合国内生成式 AI 监管要求。                    | DeepSeek / Qwen / Baichuan |

### ✅ 最终选型推荐

1. **首选方案 (高性价比 + 高性能)**：**DeepSeek V3 (API)**
   * **理由**：在能力对齐 GPT-4 的前提下，成本极低（输入 1元/百万token，输出 2元/百万token），且完全兼容 OpenAI SDK，迁移成本为零。适合作为主模型处理 RAG 和脚本生成。
2. **备选方案 (私有化/数据安全)**：**Qwen2.5-72B (自部署)**
   * **理由**：如果校方/客户要求数据不出内网，则使用 vLLM 部署 Qwen2.5-72B（需高性能显卡）或 Qwen2.5-14B（消费级显卡）。

## 3. POC 验证与实施计划

### 3.1 验证环境准备

* **基准测试集**：构建 50 个典型教育场景 Case。
  * 20 个 RAG 问答（测试基于课件的回答准确性）。
  * 10 个 幻觉测试（测试无相关知识时的拒答能力）。
  * 10 个 教学脚本生成（测试生成内容的结构化和口语化）。
  * 10 个 多轮对话（测试上下文记忆）。
* **工具链**：使用 `LangChain` 或 `PromptFlow` 进行批量评测。

### 3.2 接入与部署指南 (DeepSeek V3)

由于 DeepSeek 完全兼容 OpenAI 接口，接入非常简单。

#### 步骤 1: 获取 API Key

1. 访问 [DeepSeek 开放平台](https://platform.deepseek.com/)。
2. 注册账号并创建 API Key。
3. 充值少量金额（如 50 元，足够开发测试很久）。

#### 步骤 2: 修改项目配置

在 `backend/.env` 文件中配置（若无则创建）：

```env
# 统一使用 OpenAI 客户端格式
OPENAI_API_KEY="sk-xxxxxx"  # 填入 DeepSeek API Key
OPENAI_API_BASE="https://api.deepseek.com" # DeepSeek Base URL
LLM_MODEL_NAME="deepseek-chat" # DeepSeek V3 的模型代码
```

#### 步骤 3: 代码适配 (无需修改业务逻辑)

由于 `backend/app/services/qa/service.py` 中使用了 `ChatOpenAI`，只需确保加载了环境变量即可自动适配。

```python
# backend/app/services/qa/service.py 建议修改如下以支持配置化
import os
from langchain_openai import ChatOpenAI

class QAService:
    def __init__(self):
        self.llm = ChatOpenAI(
            model=os.getenv("LLM_MODEL_NAME", "deepseek-chat"),
            openai_api_key=os.getenv("OPENAI_API_KEY"),
            openai_api_base=os.getenv("OPENAI_API_BASE"),
            temperature=0.3,
            streaming=True
        )
        # ... 其他代码
```

### 3.3 风险缓解策略

* **API 波动风险**：实现**双模型降级策略**。
  * 主路：DeepSeek V3。
  * 备路：当主路超时/报错时，自动切换到阿里 Qwen-Turbo 或 OpenAI GPT-4o-mini（作为高可用保障）。
* **合规风险**：增加**关键词过滤层**（敏感词检测），在发送给 LLM 前和返回用户前进行双向过滤。
