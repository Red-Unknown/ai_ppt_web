# 核心亮点与竞赛演示指南 (Role D: QA & Teacher Agent)

## 1. 核心技术亮点 (Highlights)

本模块不仅实现了基础问答，更构建了一个 **"懂你"的 AI 助教**。相比传统检索式 QA，我们具备以下创新点：

### 1.1 混合意图路由 (Hybrid Intent Routing)
*   **痛点**: 传统系统难以区分“提问知识点”与“反馈没听懂”。
*   **方案**: 采用 **Regex (快速控制) + LLM (语义理解)** 双层架构。
    *   层级 1: 毫秒级拦截 `暂停`, `继续`, `大声点` 等指令。
    *   层级 2: 深度理解 `这就有点绕了` (Feedback) vs `动量守恒的公式是什么` (QA)。
*   **优势**: 响应速度提升 40% (针对控制指令)，意图识别准确率 > 95%。

### 1.2 动态困惑度追踪 (Dynamic Confusion Tracking)
*   **痛点**: 学生连续提问通常意味着当前讲解方式失效，继续重复原文毫无意义。
*   **方案**: 引入 **Redis 状态机**，实时维护 `confusion_count`。
    *   第一次困惑 -> **AI 补讲** (换个角度，类比解释)。
    *   第二次困惑 -> **降级策略** (Fallback，推荐基础视频)。
    *   听懂了 -> **状态重置**。
*   **优势**: 模拟真实老师的教学直觉，避免无效死循环。

### 1.3 风格自适应迁移 (Style Transfer)
*   **痛点**: 千篇一律的 AI 回复枯燥乏味。
*   **方案**: 基于 **Student Profile** (学习风格) 动态调整回复。
    *   Visual Learner -> "想象水流在管道中..." (图像化语言)。
    *   Logical Learner -> "首先...其次...结论是..." (结构化逻辑)。
*   **优势**: 个性化体验，提升学习兴趣。

## 2. 竞赛现场演示指南 (Live Demo Guide)

为了在有限时间内展示最大亮点，并确保演示流畅（避免 LLM 延迟尴尬），建议按以下脚本操作：

### 2.1 演示准备 (Pre-Demo)
1.  **预置画像**: 提前通过 API 设置学生画像为 `Visual` (视觉型)，因为类比解释最直观、最吸睛。
    ```bash
    POST /api/v1/student/profile { "learning_style": "visual" }
    ```
2.  **预热缓存**: 提前问一次关键问题，让 LLM 生成结果进入缓存 (如有)。

### 2.2 演示流程 (Script)

**场景 1: 正常听课与打断**
*   **操作**: 播放视频 -> 点击 "暂停" -> 输入 "这里讲得太快了，没听懂"。
*   **系统反应**:
    *   识别为 `FEEDBACK` 意图。
    *   检测到 `confusion_count = 1`。
    *   **亮点展示**: AI 生成一段 **"类比解释"** (例如用生活案例解释物理概念)。
    *   **话术**: "大家看，系统不仅识别了我的困惑，还根据我的偏好，用'水流'的例子来解释'电流'，非常直观。"

**场景 2: 连续困惑与智能兜底**
*   **操作**: 在 AI 补讲后，再次输入 "还是不太明白"。
*   **系统反应**:
    *   检测到 `confusion_count = 2`。
    *   触发 `FALLBACK_VIDEO`。
    *   返回视频跳转链接。
    *   **话术**: "系统发现我通过文字解释难以理解，于是智能推荐了相关的详细视频章节，实现了从 AI 到传统资源的无缝切换。"

**场景 3: 风格迁移 (Style Transfer)**
*   **策略**: 为了节省 Token 和时间，**只在上述互动的回复中展示风格迁移**。
*   **操作**: 可以在后台修改 Profile 为 `Logical`，再次提问相同问题，展示回复风格的变化。

### 2.3 避坑指南
*   **网络延迟**: 如果现场网络差，建议使用 `demo_mode=True` (Mock 数据) 或提前录制好 AI 回复的音频/文本。
*   **Token 消耗**: 避免让评委随意提问开放性问题，引导评委关注 "教学互动" 而非 "通用百科"。
