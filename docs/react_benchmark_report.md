## 最终答案

**递归 (Recursion)** 是一种编程技术，其中函数直接或间接地调用自身来解决问题。递归的关键特征是：

1. **自引用**: 函数在其定义中调用自身
2. **基本情况 (Base case)**: 停止递归的条件（防止无限循环）
3. **递归情况 (Recursive case)**: 函数使用修改后的参数调用自身的部分，向基本情况移动

简单来说，递归就是用自身来定义某事。数学中的阶乘函数是一个经典例子：
- factorial(0) = 1 (基本情况)
- factorial(n) = n × factorial(n-1) (递归情况)

递归特别适用于可以分解为更小、相似子问题的问题，例如树遍历、排序算法（如快速排序和归并排序）以及求解数学序列。

## **Python 中列表 (List) 和元组 (Tuple) 的关键区别**

### **1. 可变性 (最重要的区别)**
- **列表是可变的**: 创建后可以修改、添加或删除元素
- **元组是不可变的**: 一旦创建，不能更改、添加或删除元素

### **2. 语法**
- **列表使用方括号**: `[1, 2, 3]`
- **元组使用圆括号**: `(1, 2, 3)` 或仅用逗号: `1, 2, 3`

### **3. 性能**
- **元组比列表更快**: 在访问元素时
- **元组使用更少的内存**: 比列表更节省内存
- 这是因为元组是不可变的，Python 可以对其进行优化

### **4. 使用场景**
- **使用列表当**:
  - 你需要修改集合（添加、删除、更改元素）
  - 你处理的是同质数据（所有元素类型相同）
  - 你需要动态大小

- **使用元组当**:
  - 你有不应该改变的固定数据（如坐标、RGB 值）
  - 你需要数据完整性（防止意外修改）
  - 你将集合用作字典键（元组可以作为键，列表不能）
  - 你从函数返回多个值

### **5. 可用方法**
- **列表有更多方法**: `append()`, `extend()`, `insert()`, `remove()`, `pop()`, `sort()`, `reverse()` 等。
- **元组有较少方法**: 只有基本方法如 `count()` 和 `index()`

### **6. 内存和速度**
- 元组更节省内存且迭代速度更快
- 由于动态特性，列表需要更多内存

### **示例:**

```python
# 列表 - 可变
my_list = [1, 2, 3]
my_list.append(4)  # 好的
my_list[0] = 10    # 好的

# 元组 - 不可变
my_tuple = (1, 2, 3)
# my_tuple.append(4)  # 错误
# my_tuple[0] = 10    # 错误
```

### **如何选择:**
- **选择列表** 如果你需要修改集合或预先不知道大小
- **选择元组** 如果你有固定数据，需要性能，或想要确保数据完整性

元组的不可变性使其更安全地表示固定的项目集合，而列表为动态数据操作提供了灵活性。
